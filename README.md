# 📦 Pre-Sales SSI Pack — Microsoft 365 / Security / Backup

This repository illustrates **how IT pre-sales work is evolving** with the help of AI.

Following the publication *“Something Big Is Happening”* by **Matt Shumer (February 2026)**, I tested **what AI can deliver** in a **realistic IT pre-sales / SSI context**.

👉 Goal: demonstrate that a **structured, coherent, client-ready pre-sales package** can now be **pre-built in under two hours**, without replacing human expertise.

---

## 📂 Repository content

* 📐 **Microsoft 365 Architecture** *(PowerPoint)* Target architecture for a Swiss SMB (15 users): Microsoft 365 Business Premium, security, Swiss **nLPD compliance**, and Veeam backup (local disk + LTO GFS + immutable Azure copy).
* 🛠️ **Technical Procedure** *(Word & PDF)* Step-by-step documentation for **junior technicians** (tenant setup, security, Intune, Purview, Veeam, operations).
* 👥 **User Guide** *(PDF)* Simple pedagogical guide: Outlook, OneDrive, Teams, SharePoint, MFA, and security best practices.
* 🤖 **Master Prompts** *(Markdown)* The exact prompts used to drive the AI.

---

## ⏱️ Method

* **Iterative Prompting Strategy:** The package was built using **3 distinct core prompts** (Architecture, Technical, and User Guide) to ensure depth.
* **Expert Refinement:** I performed **3 to 4 manual iterations** on the Technical Procedure to ensure M365 and Veeam configurations were 100% accurate.
* **Production Time:** **~2 hours** of total work, including AI steering and technical validation (a task that usually takes **1 to 2 days** of manual drafting).
* **Human-in-the-loop:** The AI acted as a high-speed architect, while the **Pre-Sales Engineer** remained responsible for technical consistency and final pricing.

👉 AI structures and calculates — **pricing responsibility remains human**.
